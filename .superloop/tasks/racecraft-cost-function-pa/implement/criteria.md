# Code Review Criteria
Check these boxes (`- [x]`) only when true.

- [x] **Correctness**: Changes satisfy the accepted plan/requirements and preserve expected behavior.
- [x] **Safety**: No obvious security or data-integrity regressions were introduced.
- [x] **Architecture Conformance**: Changes match repo conventions and avoid needless complexity.
- [x] **Performance**: No avoidable major regressions or pathological hot paths were introduced.
- [x] **Maintainability**: Diffs are understandable, cohesive, and adequately documented where needed.
