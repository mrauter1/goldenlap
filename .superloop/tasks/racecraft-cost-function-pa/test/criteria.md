# Test Audit Criteria
Check these boxes (`- [x]`) only when true.

- [x] **Coverage Quality**: New/changed behavior is covered at appropriate levels (unit/integration/e2e as applicable).
- [x] **Edge Cases**: Failure and boundary cases for changed behavior are covered.
- [x] **Flaky Risk Control**: Tests avoid nondeterministic timing/network assumptions without stabilization.
- [x] **Regression Shield**: Critical regressions from changed modules/contracts are guarded.
- [x] **Signal Quality**: Assertions validate outcomes, not only execution paths.
